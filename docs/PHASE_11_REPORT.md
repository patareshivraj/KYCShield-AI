# KYCShield AI — Phase 11 Investigator Workbench Layer

This report summarizes the implementation of the Phase 11 Investigator Workbench. This phase successfully transitions KYCShield AI from a pure backend forensic analysis engine into an interactive, deployable product ready for human analysts, investigators, and compliance officers.

## 1. Frontend Architecture
The workbench was built using a modern React stack optimized for high performance and rapid development:
- **Framework**: React 18 with Vite and TypeScript.
- **Styling**: Tailwind CSS v4, utilizing a custom dark "cinematic" color palette optimized for extended viewing periods (to reduce eye strain for investigators).
- **Icons**: Lucide React for consistent, high-quality vector iconography.
- **Routing**: React Router DOM for SPA navigation across the investigation flow.

## 2. API Integration Layer
To bridge the gap between the FastAPI backend and the React frontend, a new set of dedicated REST endpoints was introduced under `backend/app/api/endpoints/investigation.py`:
- `GET /api/v1/investigation/applicants`: Lists all historical processing jobs.
- `GET /api/v1/investigation/applicants/{id}`: Aggregates the applicant's documents and rolls up their Document-Level Risk Scores.
- `GET /api/v1/investigation/documents/{id}`: Fetches the deep-dive intelligence, including the 0-100 Risk Score, the `investigator_summary`, OCR Fields, and Evidence Clusters.
- `GET /api/v1/investigation/documents/{id}/image`: Streams the raw normalized image binary from Phase 2 to the browser.

## 3. The Visual Forensics Canvas
The core technical achievement of Phase 11 is the `ForensicCanvas` component.
Rather than forcing analysts to look at raw coordinate JSON arrays, the system uses the HTML5 `<canvas>` API to visually overlay the intelligence directly onto the document.

The canvas synchronizes the physical intrinsic dimensions of the loaded image with the absolute coordinate geometries generated by Phase 4 (OCR) and Phase 9 (Evidence Fusion).
- **OCR Toggles**: Users can toggle blue bounding boxes indicating exactly where the OCR engine extracted text.
- **Evidence Overlays**: Users can toggle red bounding boxes indicating exactly where the fusion engine identified overlapping ELA, Noise, and Compression anomalies.

## 4. Explainability UX
As mandated in Phase 10, the dashboard answers "Why?" within seconds. The `DocumentPage` layout places the Visual Overlay side-by-side with:
1. **Risk Drivers**: High-level textual explanations (e.g., "Multiple forensic signals overlap PAN Number field").
2. **Critical Clusters**: The ranked list of the exact forensic intersections that triggered the alert, including their point contribution values.
3. **Risk Factors**: The mapped categorical risks (e.g., `identity_field_tampering_risk`).

## 5. Testing and Demo Mode
The UI includes a "Demo Mode" button on the Upload screen. When clicked, it automatically pulls the `ai_pan`, `ai_aadhaar`, and `ai_bank_statement` data generated by the `test_risk.py` pipeline, allowing stakeholders to immediately experience the forensic engine's power without needing to manually upload files.

Phase 11 is complete. KYCShield AI is now an end-to-end product.
